The Pagination component provides navigation controls for paginated data. It displays the current page, total pages, and previous/next buttons with automatic disabling at boundaries.

## Visual Appearance

The Pagination renders with:
- Horizontal layout with Previous/Next buttons
- Current page indicator in the center
- Disabled state styling for buttons at boundaries
- Small text size with gray coloring
- Hover effects on enabled buttons

## Props

<ParamField path="pagination" type="object" required>
  Pagination metadata object containing:
  - `current_page`: Current page number (1-indexed)
  - `last_page`: Total number of pages
  
  Returns null if pagination is not provided or last_page ≤ 1
</ParamField>

<ParamField path="onPageChange" type="function" required>
  Callback function invoked with the new page number when navigation buttons are clicked
</ParamField>

## Behavior

### Automatic Hiding

The component automatically returns `null` (doesn't render) when:
- No pagination object is provided
- `last_page` is 1 or less (single page of data)

### Button States

- **Previous Button**: Disabled when `current_page === 1`
- **Next Button**: Disabled when `current_page === last_page`
- Disabled buttons have reduced opacity (40%)
- Enabled buttons show hover effects

### Page Navigation

- Previous button calls: `onPageChange(current_page - 1)`
- Next button calls: `onPageChange(current_page + 1)`
- No direct page number input (sequential navigation only)

## Common Use Cases

- **Product Lists**: Navigate through paginated product catalogs
- **Order History**: Browse through multiple pages of orders
- **Client Management**: Page through client databases
- **Inventory Views**: Navigate stock listings
- **Category Lists**: Browse category pages
