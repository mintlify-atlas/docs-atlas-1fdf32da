## Overview

The Clients module manages the customer database for FrontendPT. It provides full CRUD operations for client records including name, email, and phone information. Clients are used throughout the application, particularly when creating orders.

### Hook Implementation

<Tabs>
  <Tab title="Hook Source">
    ```javascript src/application/hooks/useClients.js
    import { useState, useEffect, useCallback } from "react";
    import * as clientService from "../../infrastructure/clientService";

    export function useClients() {
      const [clients, setClients] = useState([]);
      const [pagination, setPagination] = useState(null);
      const [loading, setLoading] = useState(true);
      const [error, setError] = useState(null);
      const [search, setSearch] = useState("");
      const [page, setPage] = useState(1);
      const [allClients, setAllClients] = useState([]);

      const fetchClients = useCallback(async () => {
        try {
          setLoading(true);
          const data = await clientService.getClients(page, search);
          setClients(data.clients.data);
          setPagination(data.clients);
        } catch (err) {
          setError("Error al cargar clientes");
        } finally {
          setLoading(false);
        }
      }, [page, search]);

      useEffect(() => {
        fetchClients();
      }, [fetchClients]);

      useEffect(() => {
        clientService.getAllClients().then(setAllClients);
      }, []);

      const addClient = async client => {
        await clientService.createClient(client);
        fetchClients();
      };

      const editClient = async (id, client) => {
        await clientService.updateClient(id, client);
        fetchClients();
      };

      const removeClient = async id => {
        await clientService.deleteClient(id);
        fetchClients();
      };

      return {
        clients,
        allClients,
        pagination,
        loading,
        error,
        search,
        setSearch,
        page,
        setPage,
        addClient,
        editClient,
        removeClient,
      };
    }
    ```
  </Tab>
  <Tab title="Service Layer">
    ```javascript src/infrastructure/clientService.js
    import api from "./api";

    export const getClients = async (page = 1, search = "") => {
      const params = { page };
      if (search) params.search = search;
      const { data } = await api.get("/clients", { params });
      return data;
    };

    export const getAllClients = async () => {
      const { data } = await api.get("/clients/all");
      return data.clients;
    };

    export const createClient = async client => {
      const { data } = await api.post("/clients", client);
      return data.client;
    };

    export const updateClient = async (id, client) => {
      const { data } = await api.put(`/clients/${id}`, client);
      return data.client;
    };

    export const deleteClient = async id => {
      await api.delete(`/clients/${id}`);
    };
    ```

### Editing Client Information

```javascript
const openEdit = client => {
  setEditing(client);
  setForm({ 
    name: client.name, 
    email: client.email, 
    phone: client.phone 
  });
  setFormError("");
  setShowModal(true);
};

const handleSubmit = async () => {
  if (!form.name || !form.email || !form.phone) {
    setFormError("Todos los campos son obligatorios");
    return;
  }
  try {
    if (editing) {
      await editClient(editing.id, form);
      showToast("Cliente actualizado correctamente");
    }
    setShowModal(false);
  } catch (err) {
    const msg = err.response?.data?.message ?? "Error al guardar";
    setFormError(msg);
    showToast(msg, "error");
  }
};
```

## Data Structure

### Client Object

```typescript
interface Client {
  id: number;
  name: string;
  email: string;
  phone: string;
}
```

### API Endpoints

| Method | Endpoint | Parameters | Description |
|--------|----------|------------|-------------|
| GET | `/clients` | `page`, `search` | Paginated client list |
| GET | `/clients/all` | - | All clients (for dropdowns) |
| POST | `/clients` | Client object | Create new client |
| PUT | `/clients/:id` | Client object | Update client |
| DELETE | `/clients/:id` | - | Delete client |

## Form Validation

<Warning>
  All three fields are required when creating or editing a client:
  - **Name**: Cannot be empty
  - **Email**: Must be a valid email format
  - **Phone**: Cannot be empty
</Warning>

## Integration Points

### Orders Module

Clients are primarily used in the Orders module:

```javascript
// In OrdersPage.jsx
const { allClients } = useClients();

// Client dropdown in order form
<select value={clientId} onChange={e => setClientId(e.target.value)}>
  <option value="">Selecciona un cliente</option>
  {allClients.map(c => (
    <option key={c.id} value={c.id}>
      {c.name}
    </option>
  ))}
</select>
```

<Tip>
  Use `allClients` instead of `clients` for dropdowns to avoid pagination issues.
</Tip>

## Deletion Constraints

<Warning>
  Deleting a client may fail if they have associated orders. The backend enforces referential integrity and returns an error message.
</Warning>

```javascript
const handleDelete = async id => {
  if (!confirm("¿Eliminar este cliente?")) return;
  try {
    await removeClient(id);
    showToast("Cliente eliminado correctamente");
  } catch (err) {
    const msg = err.response?.data?.message ?? "No se pudo eliminar";
    showToast(msg, "error");
  }
};
```

## Performance Considerations

- Server-side pagination handles large client databases efficiently
- Separate `allClients` endpoint prevents pagination in dropdowns
- Search queries are debounced through the useCallback dependency
- Loading states prevent multiple simultaneous requests
