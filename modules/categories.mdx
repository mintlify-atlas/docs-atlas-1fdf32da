## Overview

The Categories module provides a simple yet powerful system for organizing products into groups. It features a streamlined interface with inline creation and modal-based editing, designed for quick category management.

## User Interface

The Categories page (`src/ui/pages/categories/CategoriesPage.jsx`) features:

- **Inline Create Form**: Quick category creation without modal dialogs
- **Category Table**: Simple table with ID, Name, and Actions columns
- **Edit Modal**: Modal dialog for updating category names
- **Toast Notifications**: Feedback for all operations

The `useCategories` hook manages category state with client-side pagination:

```javascript
import { useCategories } from "../../../application/hooks/useCategories";

function MyComponent() {
  const {
    categories,           // All categories

    catPagination,        // Pagination metadata
    setCatPage,           // Change page
    loading,              // Loading state
    error,                // Error message
    addCategory,          // Create category
    editCategory,         // Update category
    removeCategory        // Delete category
  } = useCategories();

  return (
    // Your component JSX
  );
}
```

### Hook Implementation

<Tabs>
  <Tab title="Hook Source">
    ```javascript src/application/hooks/useCategories.js
    import { useState, useEffect } from "react";
    import * as categoryService from "../../infrastructure/categoryService";

    export function useCategories() {
      const [categories, setCategories] = useState([]);
      const [loading, setLoading] = useState(true);
      const [error, setError] = useState(null);
      const [catPage, setCatPage] = useState(1);
      const itemsPerPage = 10;

      const fetchCategories = async () => {
        try {
          setLoading(true);
          const data = await categoryService.getCategories();
          setCategories(data);
        } catch (err) {
          setError("Error al cargar categorías");
        } finally {
          setLoading(false);
        }
      };

      useEffect(() => {
        fetchCategories();
      }, []);

      const addCategory = async name => {
        const newCat = await categoryService.createCategory(name);
        setCategories(prev => [...prev, newCat]);
      };

      const editCategory = async (id, name) => {
        const updated = await categoryService.updateCategory(id, name);
        setCategories(prev => prev.map(c => (c.id === id ? updated : c)));
      };

      const removeCategory = async id => {
        await categoryService.deleteCategory(id);
        setCategories(prev => prev.filter(c => c.id !== id));
      };

      // Client-side pagination
      const paginatedCategories = categories.slice(
        (catPage - 1) * itemsPerPage, 
        catPage * itemsPerPage
      );

      const catPagination = {
        current_page: catPage,
        last_page: Math.ceil(categories.length / itemsPerPage),
      };

      return { 
        categories, 
        paginatedCategories, 
        catPagination, 
        setCatPage, 
        loading, 
        error, 
        addCategory, 
        editCategory, 
        removeCategory 
      };
    }
    ```
  </Tab>
  <Tab title="Service Layer">
    ```javascript src/infrastructure/categoryService.js
    import api from "./api";

    export const getCategories = async () => {
      const { data } = await api.get("/categories");
      return data.categories;
    };

    export const createCategory = async name => {
      const { data } = await api.post("/categories", { name });
      return data.category;
    };

    export const updateCategory = async (id, name) => {
      const { data } = await api.put(`/categories/${id}`, { name });
      return data.category;
    };

    export const deleteCategory = async id => {
      await api.delete(`/categories/${id}`);
    };
    ```

```javascript
const openEdit = category => {
  setEditingCategory(category);
  setEditingName(category.name);
  setEditError("");
  setShowModal(true);

const handleEdit = async () => {
  if (!editingName.trim()) return;
  try {
    await editCategory(editingCategory.id, editingName.trim());
    setShowModal(false);
    showToast("Categoría actualizada correctamente");
  } catch (err) {
    const msg = err.response?.data?.message ?? "Error al editar categoría";

    showToast(msg, "error");

};
```

### Deleting a Category

<Warning>
  Deleting a category may fail if it has associated products. The backend validates this constraint and returns an appropriate error message.
</Warning>

```javascript
const handleDelete = async id => {
  if (!confirm("¿Eliminar esta categoría?")) return;
  try {
    await removeCategory(id);
    showToast("Categoría eliminada correctamente");
  } catch (err) {
    const msg = err.response?.data?.message ?? "No se pudo eliminar";
    showToast(msg, "error");
  }
};
```

## Architecture Decision: Client-Side Pagination

Categories use client-side pagination for two important reasons:

1. **Dropdown Usage**: The Products module needs all categories for its dropdown
2. **Small Dataset**: Categories are typically few, making client-side pagination efficient

```javascript
// All categories fetched once
const { data } = await api.get("/categories");
return data.categories;

// Paginated in the hook
const paginatedCategories = categories.slice(
  (catPage - 1) * itemsPerPage, 
  catPage * itemsPerPage
);
```

## Error Handling

Categories handle two main error scenarios:

1. **Empty Name Validation**: Checked client-side before API call
2. **Backend Constraints**: Such as deleting categories with associated products

```javascript
const handleCreate = async () => {
  if (!newName.trim()) return;  // Client validation
  try {
    await addCategory(newName.trim());
    setNewName("");
    setCreateError("");
  } catch (err) {
    const msg = err.response?.data?.message ?? "Error al crear categoría";
    setCreateError(msg);  // Display inline
    showToast(msg, "error");  // Also show toast
  }
};
```
