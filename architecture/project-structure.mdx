# Project Structure

This page provides a comprehensive overview of how FrontendPT organizes its codebase. Understanding this structure is essential for navigating the project and contributing effectively.

## Root Structure

```
src/
├── domain/              # Business entities and types
├── application/         # Use cases and application logic
├── infrastructure/      # External services and API clients
├── ui/                 # User interface layer
├── assets/             # Static assets (images, icons)
├── App.jsx             # Root application component
├── App.css             # Global app styles
├── main.jsx            # Application entry point
└── index.css           # Global CSS styles
```

## Directory Breakdown

### Domain Layer (`src/domain/`)

The domain layer contains pure business logic and type definitions.

```
domain/
└── types.js            # Domain entity factory functions
```

**Purpose**: Define the shape of business entities independent of any framework or external service.

**Key Exports**:
- `createProduct(data)` - Product entity factory
- `createCategory(data)` - Category entity factory
- `createClient(data)` - Client entity factory

**Example** from `src/domain/types.js:3`:
```javascript
export const createProduct = data => ({
  id: data.id,
  name: data.name,
  price: parseFloat(data.price),
  stock: data.stock,
  category_id: data.category_id,
  category: data.category ?? null,
});
```

### Application Layer (`src/application/`)

Contains application-specific business rules and orchestration logic.

```
application/
└── hooks/
    ├── useProducts.js      # Product management logic
    ├── useCategories.js    # Category management logic
    ├── useClients.js       # Client management logic
    ├── useInventory.js     # Inventory management logic
    ├── useOrders.js        # Order management logic
    └── useToast.js         # Toast notification logic
```

**Purpose**: Encapsulate use cases and coordinate between infrastructure services and UI components.

**Hook Responsibilities**:
- State management for specific domains
- Orchestrating CRUD operations
- Handling loading and error states
- Managing pagination and search

**Example** from `src/application/hooks/useProducts.js:4`:
```javascript
export function useProducts() {
  const [products, setProducts] = useState([]);
  const [pagination, setPagination] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);
  const [search, setSearch] = useState("");
  const [page, setPage] = useState(1);

  // Fetch products from infrastructure layer
  const fetchProducts = useCallback(async () => {
    try {
      setLoading(true);
      const data = await productService.getProducts(page, search);
      setProducts(data.products.data);
      setPagination(data.products);
      setError(null);
    } catch (err) {
      setError("Error al cargar productos");
    } finally {
      setLoading(false);
    }
  }, [page, search]);

  return {
    products,
    pagination,
    loading,
    error,
    search,
    setSearch,
    page,
    setPage,
    addProduct,
    editProduct,
    removeProduct,
  };
}
```

### Infrastructure Layer (`src/infrastructure/`)

Handles communication with external systems and services.

```
infrastructure/
├── api.js                  # Axios instance configuration
├── productService.js       # Product API endpoints
├── categoryService.js      # Category API endpoints
├── clientService.js        # Client API endpoints
├── inventoryService.js     # Inventory API endpoints
└── orderSrevice.js         # Order API endpoints
```

**Purpose**: Abstract external dependencies (APIs, databases) behind service interfaces.

**Base API Configuration** from `src/infrastructure/api.js:5`:
```javascript
const api = axios.create({
  baseURL: "http://localhost:8000/api",
  headers: {
    "Content-Type": "application/json",
    Accept: "application/json",
  },
});
```

**Service Pattern** from `src/infrastructure/productService.js:3`:
```javascript
// Each service exports CRUD operations
export const getProducts = async (page = 1, search = "", category_id = "") => {
  const params = { page, search };
  if (category_id !== "") params.category_id = category_id;
  const { data } = await api.get("/products", { params });
  return data;
};

export const createProduct = async product => {
  const { data } = await api.post("/products", product);
  return data.product;
};

export const updateProduct = async (id, product) => {
  const { data } = await api.put(`/products/${id}`, product);
  return data.product;
};

export const deleteProduct = async id => {
  await api.delete(`/products/${id}`);
};
```

## File Naming Conventions

### Components & Pages
- **PascalCase** for component files: `ProductsPage.jsx`, `Modal.jsx`
- **PascalCase** for component names matching file names

### Services & Utilities
- **camelCase** for service files: `productService.js`, `api.js`
- **Named exports** for service functions: `export const getProducts = ...`

### Hooks
- **camelCase** with `use` prefix: `useProducts.js`, `useToast.js`
- **Named exports** for hook functions: `export function useProducts() { ... }`

### Types & Domain
- **camelCase** for files: `types.js`
- **camelCase** with `create` prefix for factories: `createProduct`, `createCategory`

## Module Pattern

Each business domain follows a consistent vertical slice:

```
Products Module:
  domain/types.js                    → createProduct()
  infrastructure/productService.js   → API calls
  application/hooks/useProducts.js   → Business logic
  ui/pages/products/ProductsPage.jsx → User interface

Categories Module:
  domain/types.js                      → createCategory()
  infrastructure/categoryService.js    → API calls
  application/hooks/useCategories.js   → Business logic
  ui/pages/categories/CategoriesPage.jsx → User interface
```

## Adding New Features

When adding a new feature, follow this structure:

1. **Define domain type** in `domain/types.js`
2. **Create service** in `infrastructure/[feature]Service.js`
3. **Implement hook** in `application/hooks/use[Feature].js`
4. **Build UI** in `ui/pages/[feature]/[Feature]Page.jsx`
5. **Add route** in `App.jsx`

<Tip>
This vertical slice approach ensures all layers are properly implemented for each feature.
</Tip>
